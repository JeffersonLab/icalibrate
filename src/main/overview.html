The iCalibrate application is for operators at JLab, and assists in ion chamber calibration.

Usage

The iCalibrate application is distributed as an executable jar file. The iCalibrate application requires Java 8 or greater and can be launched on Linux with the following command: 


./icalibrate.sh
Or to have debugging logged to the console: 

./debug-icalibrate.sh
Or to invoke Java manually (without the bash shell's Java version check): 

java -jar icalibrate.jar

The versioning is organized into "major.minor" where major versions indicate API changes and minor versions are for bug fixes and implementation changes.

Configuration

A Java properties file named icalibrate.properties is located in the config directory. Users can set various configuration parameters there. There is also an epics.properties file which instructs iCalibrate how to connect to EPICS.

Source Code

The Java iCalibrate application is named icalibrate and is stored in the git repository on devl00. It can be accessed by members of the JLab accelerator group epics from an accelerator Linux machine with the command:


git clone ssh://devl00/usr/devsite/git/icalibrate.git

An Apache Ant build file (build.xml) is used to compile and package the application.
